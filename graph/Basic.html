Basic of Graph

Introduction to Graphs

A graph is a non-linear data structure consisting of vertices (nodes) and edges. The fundamental components of a graph are:

  • Vertex (Node): A point or object in the graph.
  • Edge: A connection between two vertices.

Graphs are non-linear because they allow multiple paths between vertices, unlike linear data structures such as arrays or linked lists.

Types of Graphs

Directed Graphs (Digraphs)

In a directed graph, each edge has a direction, going from one vertex to another. This direction is typically represented by an arrow. Directed graphs are used in scenarios such as:

  • Task Scheduling: Tasks are vertices, and dependencies are directed edges.
  • Web Page Link Analysis: Web pages are vertices, and hyperlinks are directed edges.

Undirected Graphs

In an undirected graph, edges have no direction. The connection between two vertices is bidirectional. Undirected graphs are commonly used in:

  • Social Networks: Relationships between individuals are bidirectional.
  • Network Topologies: Computer networks where connections are mutual.

Graph Terminology

  • Path: A sequence of edges connecting a series of vertices. For example, in a graph representing a city map, a path could represent a route from one location to another.
  • Loop: An edge that connects a vertex to itself. In a network, this might represent a feedback loop or self-referencing connection.
  • Degree: The number of edges incident to a vertex. In social networks, the degree of a vertex could represent the number of friends an individual has.
  • Cycle: A path that starts and ends at the same vertex without repeating any edge. In logistics, cycles might represent routes that return to the starting point.
  • Connected Graph: A graph where there is a path between any pair of vertices. Useful in scenarios like ensuring all nodes in a network are reachable.
  • Complete Graph: A graph in which every pair of distinct vertices is connected by a unique edge. This is used in scenarios requiring maximum connectivity, like communication networks.
  • Weighted Graph: A graph where edges have weights representing values such as distance or cost. Used in routing problems, like finding the shortest path in a map.
  • Simple Graph: A graph without loops or multiple edges between the same pair of vertices. Simple graphs are often used in theoretical studies and algorithms.

Graph Representations

Graphs can be represented in various ways, each with its own advantages:

  • Adjacency Matrix: A 2D array where each cell at index (i, j) represents the presence or weight of an edge from vertex i to vertex j.
  • Adjacency List: An array where each vertex points to a list of adjacent vertices. This representation is more space-efficient for sparse graphs.

In this tutorial, we'll use the Adjacency Matrix representation for simplicity and broad applicability. You'll learn how to implement it on the next page.
